## 1. 汇报内容概要

### 1.1 炒茄子汇报要点

1. **VAI 替代旧指标**: VAI 以 0 为分界线, 值域 [-200, 200], 两端对称, 比旧指标的 [-1, 1] 更适合阈值分析. (L9)

2. **Buffer 4km 方法**:
   - 基于 DEM 提取河道中心线
   - 做 4km 缓冲区 (实际上是河道每侧 4km, 总宽 ~8km)
   - 缓冲区范围内计算 3km 格网的 VAI
   - MIN_PIXEL_THRESHOLD = 50 (任一侧有效像元 < 50 → NaN)

3. **核心发现**:
   - 绝对海拔反转: 整体 ~3700m (VAI > 0 迎风坡更绿 → VAI < 0 背风坡更绿)
   - 相对河道高差反转: 整体 ~1400m, 比绝对海拔更明显
   - 四河相对高差阈值集中在 1300–1500m; 绝对海拔离散度大. (L13)

4. **金沙江问题**: bootstrap 500 次中仅 ~20 次检测到阈值反转, 可靠性极低. (L14)

5. **待探索现象**: 部分河谷在 ~2km 处 VAI 先降低后抬升 (双峰). (L15)

## 2. 赵老师反馈 — 逐条详细

### 2.1 核心方法学批评: "平均抵消效应"

> "你现在做的是一个平均效应, 会综合掉, 会把它的这样的一个剧烈的差异给综合掉." — 赵伟 (L253)

**问题描述:**
- 当前 3km 格网内, 各像元的高程范围覆盖从谷底到山顶
- 低海拔: 迎风坡 NDVI 远高于背风坡 → VAI 强正
- 高海拔: 背风坡 NDVI 反超迎风坡 → VAI 为负
- 两者在同一个网格内平均, 正负抵消, VAI 被大幅压缩

**赵老师建议的方案:** (L149, L257)

> "你可以设置在比如说距离河面是500m, 我只算这个500m的差...你可以做200m 300m 400m 500m 的这样的一个不同的这个海拔高度, 我就做这个这样的一些范围, 我就可以算出这个 VAI 来, 我可能 VAI 就是一个保持的一个比较大的值."

**具体操作**: 对每个 3km 格网, 不取全部像元, 而是只取距离最近河道高度差 ≤ H 的像元 (H = 200m, 300m, 400m, 500m, 1000m 等), 每个 H 单独算一个 VAI.

**预期效果**: VAI 绝对值会增大, 反转点可能更清晰, 金沙江的不稳定性可能改善.

### 2.2 网格位置敏感性 (Placement Bias)

> "我这个网格, 我往这边偏一点, 往那边偏一点, 它的值就会带来很大的影响." — 赵伟 (L213)

**问题描述:**
- 3km 格网是全局统一划分的, 非河道中心对齐
- 某网格恰好覆盖河道两岸的迎风坡+背风坡 → 算得准
- 同一网格若偏移一点, 可能只覆盖一侧或河谷外 → 算不准
- 这不是"尺度效应", 而是与河道空间关系不对齐造成的位置偏差

**赵老师建议:** (L165, 后文详细展开)

> "你可以测试1km 2km 3km 这样的一些网格就是这样的一个 buffer"

做 1km / 2km / 3km 的多尺度对照, 看结果是否对网格大小鲁棒.

### 2.3 网格定义方式: 应相对于河道, 而非全局划分

> "你根本不有去讨论这个 buffer 你你这就是把我们的这样的一个川西的研究区, 我就直接把它本来是10m×10m 的, 我再把它重新重采样, 或者说按照这个三3km×3km 的网格去重新做了." — 赵伟 (L205)

**关键区分:**
- **当前做法**: 对全川西研究区画统一的 3km 网格 → 再选与河道 buffer 相交的格网
- **赵老师期望**: 以河道中心线为基准, 沿河道做 buffer, buffer 内部再做格网划分

> "你其实...没有去讨论他的这样一个 buffer 的一个一些情况, 你根本没有去讨论 buffer." (L205)

### 2.4 空间分布图的边界完整性

> "你大区的图不要做这样的一个人为的边界, 因为这里面是金沙江就已经就在这一块了, 你应该做的图是一个一个比较完整的, 能够涵盖整个区域的." — 赵伟 (L307)

当前空间分布图在金沙江跨出川西处被人为截断. 应做到完整覆盖研究区、无人工边界切割, 尤其是金沙江源头和下游延伸段.

## 4. 未来研究方向 (赵老师提议)

### 4.1 沿河段转折点制图 (空间异质性)

> "下面还有一种分析方式是什么？是要比如说沿着这条河, 我们就可以做它的一个 buffer 的一个制图, 比如说它的这样, 这个转折点的制图. 比如说这边可能是一多少米, 这边是多少米...你就可以沿着这条河...这在这一段, 它的这样的一个转折点...我每一个河段, 我们都可以算出一个一个值来, 就是相当于它的这样的一个转折的这个点的空间分布." — 赵伟 (L293–L297)

**具体含义:**
- 不是做一个整条河的平均 VAI 阈值, 而是对每个河段 (如沿河每 5km 或 10km) 单独计算该段河道 buffer 内 VAI 的绝对/相对反转点
- 制作"沿河段的 VAI 阈值空间分布图"
- 然后可以回答: 为什么这一段是 1300m, 那一段是 1500m? → 引入局地地形因子 (河谷宽度、坡度、走向、弯曲度等)

这与 `zhao_threshold_intensity.py` (Step 4) 理念呼应, 但更聚焦于**反转点的空间映射**而非单纯的 |VAI| 强弱分级.

### 4.2 干旱河谷边界重定义 (先验知识 + 山脊线)

> "但是有了这样的一个, 这是相当于是一个先验知识, 然后这个先验知识是怎么再去指导我们去做...去制图, 是不是, 有了这样的先验知识, 那我们就有了先验知识, 我们就可以去什么去沿着河道去做这样的一个山脊线." — 赵伟 (L273)

> "你就沿着河道两岸我们是不是就有它的山脊线了, 那我们就沿着山脊线, 我们就可以把它整个干旱河谷全都给画出来. 这是在我们的这样的一个先验知识指导下的这样的一个干旱河谷的这样的一个重新的整个西南地区的干旱河谷的重新的定义." (L277)

**完整逻辑链:**
1. VAI 空间分布 → 识别河谷核心影响带 (现有, 已有初步结果)
2. 河谷核心带作为"先验知识"
3. 沿河道两岸提取山脊线
4. 河谷边界 = 河道 + 两侧山脊线所围区域
5. 输出: 基于遥感 VAI 的西南干旱河谷新边界

**意义**: 这是从"统计反转阈值"升格为"方法论创新 + 制图产品"的关键一步.

### 4.3 推广到西南其他干旱河谷

炒茄子提到 "也在尝试一下其他西南地区它还有好几条干旱河谷的区域...看这是有一个通用性的规律" (L15). 赵老师没有反对, 但提醒先解决当前方法学问题.

## 6. 当前方法 vs. 赵老师期望 — 对照表

| 维度 | 当前做法 | 赵老师期望 | 影响 |
|------|---------|-----------|------|
| VAI 高度范围 | 格网内全部像元 (谷底→山顶) | 仅取距河道特定高度差 H 以内的像元 | VAI 值增大, 反转点可能变化 |
| 网格定义 | 全局 3km 网格, 后选与 buffer 相交的 cell | 沿河道 buffer 内部做网格 (或至少以河道为中心) | 消除位置偏差 |
| 网格 size | 仅 3km | 1km / 2km / 3km 对照 | 验证鲁棒性 |
| 分析粒度 | 整河谷平均 | 沿河段分段空间制图 | 回答"为什么不同河段不同" |
| 边界定义 | 基于 polygon / buffer | 先验知识 (VAI) + 山脊线 | 方法学创新 |
| 空间分布图 | 有人为边界 (川西截断) | 完整研究区覆盖 | 出图专业性 |
