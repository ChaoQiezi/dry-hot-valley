## 3. 生长季重算: 完整链条

### 3.1 动机

ERA5 生长季从 6-8 月 → 5-9 月. 原因: 川西干热河谷植被生长季始于 5 月; 5-9 月能更好代表雨季 + 生长旺季的盛行风向.

### 3.2 链条 (Step 0 → 5, 全部已重跑)

```
Step 0: ERA5 nc (I盘) → 0.25° u/v/wind_dir (G盘, 5-9月加权平均)
Step 1: 0.25° u/v → 10m u/v (重投影+重采样, G盘)
Step 2: 10m u/v → 10m u/v masked (DEM 掩膜, G盘)
Step 3: masked u/v → 10m wind_dir (4 气压层, G盘)
Step 4: 600hPa wind_dir + DEM → wind_effect.tif (SAGA GIS, G盘, ~数小时)
Step 5: wind_effect + slope → windward_leeward.tif (二分类, 1=迎风, 2=背风, 255=nodata/slope≤5°)
```

**管道脚本**: `Src/preprocessing/run_growing_season_pipeline.py`

### 3.3 windward_leeward_divide.py Bug 修复记录 (Step 5)

| Bug | 原因 | 修复 |
|-----|------|------|
| `predictor=3` 不兼容 uint8 | 机械复制 float32 脚本的 TIFF profile | → `predictor=2` (整数 predictor) |
| block 尺寸非 16 倍数 | SAGA 输出 striped (1-pixel blockysize), 复制其 profile 时继承了不兼容的 block 尺寸 | → `blockxsize=4096, blockysize=4096` |
| 损坏的 windward_leeward.tif | 前次失败运行残留, GDAL 在 open 时读取损坏 metadata | 删除旧文件 → 重新运行 |

这些 bug 均在 Step 5 (`windward_leeward_divide.py`) 中, 由用户发现并修复, Claude 审核确认.

## 5. 管道脚本设计

### 5.1 run_growing_season_pipeline.py (预处理链)

```
python_geo: D:/Softwares/Anaconda3/envs/geo/python.exe
cwd: Src/preprocessing/
顺序: Step 0 → 1 → 2 → 3 → 4 → 5 (严格依赖, 上游失败立即停止)
force_rerun: True
```

各步骤调用子脚本 via `subprocess.run()`, 检查输出文件是否存在.

### 5.2 run_vai_buffer_pipeline.py (VAI 分析链)

```
python_geo:  D:/Softwares/Anaconda3/envs/geo/python.exe       (Step 2-4)
python_arcpy: D:/Softwares/Anaconda3/envs/ArcGISPro/python.exe (Step 1)
cwd: F:\PyProJect\dry_hot_valley\Src\valley_analysis
顺序: Step 1 → 2 → 3 → 4 (Step 4 注释, 可选)
```

**级联逻辑:**
- `force_rerun=True` → 全部重跑.
- `force_rerun=False` → 检查输出存在则跳过.
- `upstream_was_run` 链式标记: 上游重跑了, 下游即使输出存在也必须重跑 (保证数据一致性).
- `manual_step1_done` 标记: 当 Step 1 (unify_dataset) 在 PyCharm 中手动运行后, 设置此标记为 True, 下游会自动重跑.

**Step 1 (unify_dataset × 4):**
- 使用 ArcGISPro 解释器 (依赖 arcpy).
- 循环 4 河谷, 各调用 `{Valley}/preprocessing/unify_dataset.py`.
- 仅裁剪 `windward_leeward.tif` 到河谷范围 → `windward_leeward_region.tif`.
- DEM/NDVI 裁剪已注释 (用户确认不需要重生成).

**Step 2 (buffer VAI 3km):**
- `geo_analysis/VAI_spatial_distribution_buffer.py` — 一次运行完成四河.
- 算法: 河道中心线 4km buffer → 10m 栅格 mask → 3km 网格 VAI (MIN_PIXEL_THRESHOLD=50).
- 输出每河谷 8 个栅格 (VAI, p-value, NDVI_windward/leeward, DEM, buffer_fraction, windward_count, leeward_count).

**Step 3 (合并阈值 + 森林图):**
- `geo_analysis/VAI_altitude_gradient_buffer.py` — 四河合并 + 分河独立.
- LOWESS smooth (frac=0.35) + bootstrap (500, cell-level resample) + forest plot.
- 输出: cells_all.csv, gradient CSV × 2, threshold_summary_all.csv, threshold_all.png.

**Step 4 (河段强弱分级, 当前注释):**
- `geo_analysis/zhao_threshold_intensity.py`.
- 使用 30 km 河段邻域统计 |VAI| 中位数三档分级.
- 待用户决定是否需要重跑 (用最新 buffer 数据).

## 8. 当前不能说什么 (诚实边界)

1. **不能说**: "3652m 是焚风停止高度." → 应说: "3652m 是 4 km buffer 内 VAI 由正转负的反转海拔."

2. **不能说**: "四条河谷相对高差阈值完全一致." → 金沙江 REL 反转不稳定; 大渡河/岷江/雅砻江 REL 较接近 (~1.2-1.3 km).

3. **不能说**: "已经完整回答'有些山差异大, 有些不大'." → 河段强弱分级尚未用最新 buffer 数据重做 (Step 4 仍注释).

4. **不能说**: "NDVI 已更新为 5-9 月." → 当前 NDVI 仍是 GEE 原始 5-9 月 (2019-2025 逐年 + 多年均值). 如果用户后续改用 5-9 月特定合成, 需单独重跑 NDVI 链.

5. **不能说**: "4 km 是最优 buffer 宽度." → 未做敏感性分析 (3 km / 5 km).

## 11. 论文表述建议

### 11.1 Abstract 关键句草案

> Across four dry-hot valleys in western Sichuan, we computed the Vegetation Asymmetry Index within a 4 km river buffer at 3 km grid resolution. LOWESS smoothing and bootstrap resampling (500 iterations) reveal a consistent reversal from windward-dominated to leeward-dominated vegetation at approximately 3652 m elevation (95% CI: 3573–3757 m), equivalent to 1356 m above the nearest river channel. The reversal is robust across all four valleys combined (bootstrap support 100%), though individual valleys show elevation differences of up to 940 m, reflecting topographic and climatic heterogeneity.

### 11.2 Methods 关键声明

- "干热河谷核心影响带" = 河道中心线 4 km buffer, 非人为 polygon.
- 迎背风坡分类: SAGA Wind Effect + slope>5° 过滤.
- VAI = (NDVI_windward − NDVI_leeward) / mean(NDVI) × 100%.
- Grid cell 任一侧有效像素 < 50 → NaN (MIN_PIXEL_THRESHOLD).
- Bootstrap: cell-level 有放回重采样 500 次, LOWESS frac=0.35.
- 此分析回答 "迎背风植被差异沿海拔反转阈值", 不直接等同"焚风效应停止高度".

### 11.3 Discussion 可强调的原创性

- 国际文献 (Yin 2023, Tian 2026 NCC, Liang 2024) 未在多相邻河谷上比较 VAI 反转海拔阈值.
- 中国干热河谷对比 (Liu 2023 RS, Tang 2018 MRD) 未做坡向不对称的海拔阈值.
- 跨四河 3652m 共性与 940m 分异 → 可引出"区域气候 vs 局地地形" 的讨论.
