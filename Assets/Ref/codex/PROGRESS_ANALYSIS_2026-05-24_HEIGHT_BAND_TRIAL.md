## 0. BLUF

2026-05-17 组会指出的核心问题成立于方法层面：现行 `3 km` 网格 VAI 先将谷底到山体高处的像元合并，再计算迎/背风差异，确实可能稀释近河谷底的强信号。

本次只以岷江做方法试验，得到两个需要分开的结论：

1. **已观察到稳定的振幅稀释。** 与同一网格的全高度 VAI 相比，限制到相对河床 `0-300 m`、`0-400 m`、`0-500 m` 后，配对 `|VAI|` 中位增量分别为 `+3.05`、`+2.20`、`+1.48` 个百分点，bootstrap 95% CI 均高于 `0`。
2. **尚未观察到岷江的正负抵消。** 互斥高度带的 VAI 中位数由近河 `0-200 m` 的 `11.88%` 降到 `1000-1500 m` 的 `0.64%`，但到 `1500-2000 m` 仍为正；这更支持“高处弱信号稀释近河强信号”，不能写成“高处负 VAI 与低处正 VAI 发生抵消”。

本试验使用临时河道定义和历史完整输入，只用于检验方法方向，不替代后续正式结果。

## 1. 问题定义修正

组会提出的“距河道 200/300/400/500/1000 m”在方法上应解释为**相对河床高程**，不是水平 buffer 半径：

```text
relative_height(pixel) =
    DEM(pixel) - DEM(nearest sampled river-channel point)
```

现有主线脚本 `VAI_spatial_distribution_buffer.py` 仅执行：

```text
4 km 水平河道 buffer -> 3 km 网格内所有有效像元 -> 一个 VAI
```

它没有在计算 VAI 前用逐像元相对河床高程分层。因此，旧的 `VAI_3km_buffer.tif` 不能回答平均抵消问题。

## 3. 方法

固定不变的旧方法部分：

- 水平河道 buffer: `4 km`
- 统计网格: `3 km`
- `NDVI > 0.1`
- 任一迎/背风侧有效像元 `< 50` 时该网格高度层无效

新增的平均抵消诊断部分：

1. 将候选河道以 `10 m` 步长采样，并从 DEM 读取河床高程。
2. 对 buffer 内每个有效像元查询最近河道采样点，计算逐像元相对河床高程。
3. 在同一 `3 km` 网格内分别计算：
   - 全高度基线: `full_buffer`
   - 累计高度帽: `0-200/300/400/500/1000 m`
   - 互斥诊断带: `0-200`, `200-300`, `300-400`, `400-500`, `500-1000`, `1000-1500`, `1500-2000`, `>=2000 m`
4. 对累计高度帽与该网格的全高度基线进行配对比较。
5. 对高度带 VAI 中位数和配对 `|VAI|` 增量做 `1000` 次网格级 bootstrap CI。

## 5. 结论边界

### 确定知识

- 现有 VAI 主线脚本没有在网格内按逐像元相对河床高程分层。
- 在本次岷江候选河道试验中，`0-300/400/500 m` 高度帽均显著增强配对 `|VAI|`，证明全高度聚合会稀释近河 VAI 信号。
- 岷江互斥高度带 VAI 从近河向高处分层减弱。

### 推断

- 对岷江而言，“平均抵消效应”的更准确表述目前是**强信号被弱信号稀释**，而非正负符号直接相消。
- `300-500 m` 可作为下一轮正式岷江敏感性分析的优先高度帽范围。

### 尚不能声称

- 不能声称岷江在高高度带出现负 VAI 或证明了符号反转。
- 不能将本轮数字视作正式西南结果，因为河道基准尚未人工确认，且输入使用的是历史岷江完整数据。
- 不能用本轮试验否定旧的整河谷反转结果；两者的估计对象不同，旧结果是按网格平均高度组织的 VAI 趋势，本轮是网格内部按像元高度分层的 VAI。
