## 1. 当前研究对象到底变成了什么

### 1.1 旧主线: `_region`

`_region` 是四条河谷 polygon 的外接矩形窗口。它的优点是样本量大,但问题是包含大量河谷外山体,容易被质疑:

> 你分析的是河谷附近大窗口,不一定是干热河谷本身。

5 月 5 日 Fig 1-5 的核心结果就来自这条线。

### 1.2 strict polygon

strict 方法只在干热河谷 polygon 内重算 VAI。它更“严格”,但样本量太小,高海拔段尤其不足。

以大渡河为例:

| 方法 | 有效 3 km 网格 | ABS 反转 m | bootstrap 支持 |
|---|---:|---:|---:|
| `_region` | 约 3206 | 3394 | 接近 100% |
| `_strict` | 226 | 3263 | 58.6% |
| `_buffer` 4 km | 543 | 3530 | 99.8% |

解释:

- strict 在概念上最干净,但样本不足导致阈值不稳定。
- 它适合做方法对照,不适合作主线。

### 1.3 当前主线: 4 km 河道 buffer

buffer 方法围绕河道中心线建立 4 km 缓冲区,只在这个范围内重算 3 km VAI。

它的优势:

- 不再使用外接矩形大窗口。
- 不完全依赖外部干热河谷 polygon 的边界形状。
- 样本量比 strict 大,比 `_region` 更贴近河谷。
- 可在四条河谷上统一执行。

当前最推荐表述:

> 以河道中心线 4 km 缓冲区作为干热河谷核心影响带,在该范围内重新计算 3 km VAI,并分析迎风/背风植被差异随海拔的反转阈值。

## 2. VAI 到底是什么

VAI 的公式:

```text
VAI = (NDVI_迎风坡 - NDVI_背风坡) / [(NDVI_迎风坡 + NDVI_背风坡)/2] × 100%
```

解释:

- `VAI > 0`: 迎风坡更绿。
- `VAI < 0`: 背风坡更绿。
- `VAI = 0`: 迎风坡和背风坡 NDVI 相等。

所以“VAI 反转阈值”的含义是:

> 随海拔或相对河道高差增加,迎风坡优势逐渐减弱,到某个高度后变成背风坡优势。

注意:

- 这不是直接测量焚风。
- 它是用植被绿度差异间接刻画迎风/背风坡效应。
- 因此论文中应写“迎背风植被差异反转阈值”,而不是“焚风停止阈值”。

## 3. 4 km buffer 版 3 km VAI 是怎么计算的

代码:

- `F:\PyProJect\dry_hot_valley\Src\valley_analysis\geo_analysis\VAI_buffer_valley_3km_all.py`

核心参数:

| 参数 | 数值 | 含义 |
|---|---:|---|
| `BUFFER_RADIUS_M` | 4000 | 河道中心线两侧 4 km |
| `GRID_SIZE_M` | 3000 | 3 km VAI 网格 |
| `PIXEL_SIZE_M` | 10 | NDVI/DEM 原始像元 |
| `MIN_PIXEL_THRESHOLD` | 50 | 每个 3 km 网格内迎风/背风像元各至少 50 个 |
| `NDVI_MIN` | 0.1 | 低 NDVI 像元剔除 |

每个 3 km 网格的计算过程:

1. 只保留落在 4 km buffer 内的 10 m 像元。
2. 剔除无效 NDVI、无效 DEM、`NDVI <= 0.1`。
3. 按 `windward_leeward` 分成迎风坡像元和背风坡像元。
4. 若任一侧像元少于 50 个,该 3 km 网格剔除。
5. 分别计算迎风坡 NDVI 均值和背风坡 NDVI 均值。
6. 代入 VAI 公式。
7. 同时输出:
   - `VAI_3km_buffer.tif`
   - `DEM_3km_buffer.tif`
   - `VAI_pvalue_3km_buffer.tif`
   - `NDVI_windward_3km_buffer.tif`
   - `NDVI_leeward_3km_buffer.tif`
   - `windward_count_3km_buffer.tif`
   - `leeward_count_3km_buffer.tif`
   - `valley_fraction_3km_buffer.tif`

这里比旧版更好的地方:

- 旧版 `MIN_PIXEL_THRESHOLD` 定义后没有真正使用。
- buffer 版已经真正检查迎风/背风像元数。
- 这点对方法可靠性很重要。

## 4. ABS 与 REL 是什么

当前结果同时给两个坐标:

| 名称 | 字段 | 含义 |
|---|---|---|
| ABS | `DEM_m` | 3 km 网格的绝对海拔 |
| REL | `relative_height_m` | 3 km 网格 DEM - 最近河道中心线 DEM |

用公式写:

```text
REL = 网格平均海拔 - 最近河道海拔
```

二者回答的问题不一样:

- ABS 回答: 这个反转大约发生在多少米海拔?
- REL 回答: 相对谷底/河道抬升多少米后发生反转?

赵导说“是多少高度差”,更接近 REL。

但论文中最好两个都报:

- ABS 是遥感和生态梯度最直观的横坐标。
- REL 更接近河谷地形语义。

## 5. LOWESS 是什么

LOWESS 是一种局部平滑方法。

你可以把它理解为:

> 不强行假设 VAI 和海拔是直线关系,而是在每一个海拔附近,用附近一批点拟合一个局部趋势,最后连成一条平滑曲线。

当前代码:

- `LOWESS_FRAC = 0.35`
- 含义: 每次局部拟合大致使用 35% 邻近样本点。

为什么不用直接连散点:

- 3 km VAI 网格很散。
- 每个格网受局地坡面、植被、NDVI噪声影响。
- LOWESS 可以看总体趋势。

为什么不用简单线性回归:

- VAI 随海拔可能不是直线。
- 低海拔、中海拔、高海拔可能有不同斜率。
- LOWESS 更适合先做现象刻画。

## 6. 反转阈值是怎么找的

代码逻辑:

1. 用 LOWESS 得到平滑曲线:

```text
VAI_smooth = f(elevation)
```

2. 从低海拔向高海拔扫描。
3. 找第一次从正值变成负值的位置:

```text
VAI_smooth > 0  →  VAI_smooth < 0
```

4. 用线性插值估算交点位置。

这个交点就是当前报告的“反转阈值”。

## 7. Bootstrap 和 95% CI 是什么

### 7.1 Bootstrap 做了什么

当前代码不是只算一次阈值,而是重复 500 次:

1. 从原始 3 km 网格中随机抽样,允许重复抽。
2. 抽出同样数量的网格,形成一个“模拟样本”。
3. 对这个模拟样本重新做 LOWESS。
4. 重新找 VAI 正转负的交点。
5. 记录交点。
6. 重复 500 次。

### 7.2 95% CI 怎么来

500 个 bootstrap 交点会形成一个分布。

- 中位数 = 报告的反转阈值。
- 第 2.5 百分位 = 95% CI 下限。
- 第 97.5 百分位 = 95% CI 上限。

例如整体 ABS:

```text
3735 m [3643, 3912]
```

含义:

> 在 500 次重采样中,反转点的中位值为 3735 m,其中 95% 的 bootstrap 反转点大致落在 3643-3912 m。

### 7.3 `n_boot_with_crossing` 是什么

不是每次 bootstrap 都一定能找到正转负交点。

例如金沙江 REL:

```text
37/500
```

含义:

> 500 次重采样里只有 37 次能找到 VAI 由正转负的交点。

这说明:

- 这个轴上的反转非常不稳定。
- 不应把金沙江 REL 1642 m 当成稳健结论。
- 在 PPT 和论文中必须标注“金沙江 REL 反转不稳健”。

### 7.4 这个 CI 不是万能的

这个 95% CI 表示的是:

> 由于 3 km 网格样本组成变化导致的阈值不确定性。

它不表示:

- NDVI 传感器误差。
- DEM 误差。
- 风向分类误差。
- buffer 宽度选择误差。
- LOWESS 参数选择误差。

所以后面仍需要 buffer 宽度和风向敏感性分析。
